COMPLETE v0.0.1a — Foundation

  • Pure C++ single-binary daemon with 3 listener ports (9432 WAF, 8862 UI, 8832 API)
  • WAF rule engine — 9 built-in rules, URL decoding, regex matching, threat scoring
  • FastCGI client — forwards WAF-inspected requests directly to PHP-FPM
  • Bounded thread pools — auto-sized from CPU cores and available memory
  • MySQL schema — 16 tables, seeded with rules, agents, config, admin user
  • PHP dashboard — 13 pages with dark cybersecurity theme, Chart.js analytics
  • CRUD APIs — rules create/edit/delete, user management, site toggles, profile edit
  • Site discovery — 27 nginx sites with doc_root, log paths
  • WAF enrollment flow — per-page nginx snippet generation, 3-step wizard
  • Log analysis — direct access/error log parsing, Chart.js dashboards (hourly traffic, status codes, top pages, IPs, agents, referers)
  • Self-performance monitoring — CPU, memory, threads, FDs every 10 seconds
  • Dual clocks (military + civilian), WAF mode pill, user dropdown, toast notifications

Lessons Learned

  • Single binary wins. The initial Go sidecar architecture (3 binaries, 2 languages) was scrapped in favor of one C++ daemon. Less complexity, faster iteration, no proxy loops.
  • FastCGI is the right protocol. The WAF talks directly to PHP-FPM via FastCGI unix socket. No HTTP reverse proxy needed. No nginx middleman for the upstream.
  • URL decode before matching. Encoded payloads bypass regex rules. Always decode %XX and + before inspection.
  • std::regex doesn't support (?i). Use std::regex::icase flag instead. Strip (?i) prefix from DB-stored patterns automatically.
  • MySQL NULL crashes C++ strings. Always wrap MYSQL_ROW field access with a safe() helper that returns empty string for NULL.
