Scoring

Each matched rule adds its score to the request's total. Multiple rules can match the same request — scores are cumulative. The highest-severity action wins.

  • 0 — Clean request, no rules matched
  • 1-49 — Low concern, logged for analysis
  • 50-74 — Medium concern, flagged for review
  • 75+ — High concern, threat event created (configurable threshold)
  • 90+ — Critical — blocked in active mode

URL Decoding

Query strings and paths are URL-decoded before rule matching. This catches encoded attacks like:

  • %20UNION%20SELECT → UNION SELECT
  • %3Cscript%3E → <script>
  • %3B%20wget → ; wget

Regex Notes

  • Uses C++ std::regex with ECMAScript syntax
  • Do NOT use (?i) prefix — case-insensitive flag is applied automatically
  • Rules with invalid regex patterns are skipped with an error log
  • Regex is compiled once on rule load and cached
